✅ parm 格式最終確認
====================

## 已知信息

parm 字符串片段: `87000900`

## 十進位轉換

```
87 = 0x87 = 135 (十進位)
00 = 0x00
09 = 0x09 = 9 (十進位)
00 = 0x00
```

## 數據映射

根據協議文檔和實際數據：

```
87 00 = 0x0087 = 135 (投幣，Little-Endian)
09 00 = 0x0009 = 9 (出獎，Little-Endian)
```

## 在完整 parm 中的位置

完整 parm:
```
02020a5e010002010001000201000a00000000000000830628210b000128000087000900d2005e015a00780046000e0128000000140000000000323232320a0a00
```

找到 `87000900` 的位置:

```
位置計算:
- parm 字符串是十六進位，每 2 個字符 = 1 字節
- 字符串中找到: "87000900"
- 在位置: 68-75 (字符位置)
- 字節位置: 34-37

即:
- 位置 34 = 0x87 = 135 (投幣低位)
- 位置 35 = 0x00 = 0 (投幣高位)
- 位置 36 = 0x09 = 9 (出獎低位)
- 位置 37 = 0x00 = 0 (出獎高位)
```

## 🎯 正確公式

```javascript
// 投幣 (135)
const plays = bytes[34] | (bytes[35] << 8);
// = 0x87 | (0x00 << 8)
// = 135

// 出獎 (9)
const payouts = bytes[36] | (bytes[37] << 8);
// = 0x09 | (0x00 << 8)  
// = 9
```

---

**結論: 真正的位置是 34-35 (投幣) 和 36-37 (出獎)！** ✅
