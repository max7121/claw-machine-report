## 🎯 數據位置

### 位置 33-34: `87 00`
```
Big-Endian:     (87 << 8) | 00 = 34,560 ❌
Little-Endian:  87 | (00 << 8) = 135 ✅
```

### 位置 35-36: `09 00`
```
Big-Endian:     (09 << 8) | 00 = 2,304 ❌
Little-Endian:  09 | (00 << 8) = 9 ✅
```

但您說需要 350 和 90...

### 位置 39-40: `5e 01`
```
Big-Endian:     (5e << 8) | 01 = 24,065 ❌
Little-Endian:  5e | (01 << 8) = 350 ✅ ⭐
```

### 位置 41-42: `5a 00`
```
Big-Endian:     (5a << 8) | 00 = 23,040 ❌
Little-Endian:  5a | (00 << 8) = 90 ✅ ⭐
```

## 💡 發現！

**正確位置應該是 39-42（Little-Endian）：**

```
位置 39-40: 5e 01 → 0x015e = 350 (投幣) ✅
位置 41-42: 5a 00 → 0x005a = 90 (出獎) ✅
```

## 驗證

在完整 parm 中：
```
...0128000087000900d2005e015a00...
         ↑ 位置33-36
            ↑ 位置37-40 ← 這裡是 5e 01 5a!

00d2 00 5e 01 5a 00 78 00 46 00
        ↑位置37-38 ↑位置39-40 ↑位置41-42
```

## ✅ 正確公式

```javascript
const plays = bytes[39] | (bytes[40] << 8);    // 5e | (01 << 8) = 350
const payouts = bytes[41] | (bytes[42] << 8);  // 5a | (00 << 8) = 90
```

**結論：位置應該是 39-42，而不是 36-39！**
